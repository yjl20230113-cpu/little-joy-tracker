Problem Statement
对于很多情侣或注重生活仪式感的用户来说，现有的日记或社交应用要么过于沉重复杂，要么缺乏良好的关系隔离（无法把“关于男朋友”和“关于爸妈”的记录有效分开），且无法提供深度的情感反馈。作为一名编程小白和未来的 AI 产品经理，用户希望通过 Vibe Coding（AI 辅助编程）的方式，为女朋友开发一款专属的、轻量级的、注重隐私的“小美好记录器”。该应用需具备“即开即记”的极简体验、永久的数据保存能力，并且能通过大模型（LLM）对过去的记录进行多维度的情感总结（如性格画像、心情趋势），同时以 PWA 的形式在 iPhone 上提供类似原生 App 的体验，免去繁琐的上架流程。
Solution
构建一个基于 Next.js + Supabase + Vercel 的零成本（Zero-Cost） PWA 响应式网页应用。
系统采用邮箱/密码的身份验证来保证数据的永久存储与多账号间的绝对隔离。在交互上，应用采用“进门即录入”的速记模式，用户打开应用后直接面对包含相机调用、文本输入和默认归属人（如“男朋友”）的编辑卡片。在浏览层面，提供按天聚合的时间线（Timeline），支持精细的按人员和时间跨度筛选。核心亮点是内置 AISummaryEngine 模块，能根据筛选出的一段时间内的文字内容，调用 DeepSeek/Claude 大模型，生成包含关键词提炼、心情走势、人格侧写结论和生活建议的结构化总结卡片，提供极高的情感价值。
User Stories
As a 移动端用户, I want a 可以通过浏览器“添加到主屏幕”的 PWA 应用, so that 我可以在 iPhone 上获得类似原生 App 的全屏沉浸式体验，且不需要去 App Store 下载。
As a 记录者, I want a 邮箱和密码登录界面, so that 我的记录可以永久保存在云端，并且不会被其他使用该应用的人看到。
As a 记录者, I want a 在登录后进入应用直接看到空白的“新建记录卡片”, so that 我能在情绪和灵感发生的第一时间完成记录，不需要繁琐的点击跳转。
As a 记录者, I want a 卡片顶部能默认选中我最常记录的对象（如“男朋友”）, so that 绝大多数情况下我不需要手动选择分类就能直接写内容。
As a 记录者, I want a 能够点击顶部人员名字下拉切换其他记录对象（如“爸爸”、“小猫”）, so that 我可以将不同维度的生活美好隔离管理。
As a 记录者, I want a 明显的图片上传区域，点击后能直接唤起 iPhone 的相机或相册, so that 我可以用照片定格当下的美好瞬间。
As a 记录者, I want a 必填的文本输入框来描写“事情内容”, so that 我能清晰地记录发生了什么。
As a 记录者, I want a 选填的文本输入框来描写“理由/感悟”, so that 我可以抒发为什么这件小事让我觉得美好。
As a 记录者, I want a 系统在卡片上默认显示当天的日期（精确到日）, so that 我不需要手动输入时间。
As a 记录者, I want a 能够手动修改这篇记录发生的日期, so that 我可以补录过去几天忘记记录的美好事件。
As a 记录者, I want a 点击“完成/保存”按钮后，卡片自动收起并跳转到首页列表, so that 我能得到明确的保存成功反馈并回顾历史。
As a 记录者, I want a 点击“取消”按钮后直接关闭卡片进入首页列表, so that 我可以在不想记录时快速退出编辑状态。
As a 回顾者, I want a 首页以“天”为单位的瀑布流/时间线展示记录, so that 我可以直观地浏览每一天发生的所有美好瞬间。
As a 回顾者, I want a 首页列表的卡片上显示照片缩略图和文字摘要, so that 我能快速扫视并回忆起当时的情景。
As a 回顾者, I want a 顶部有“人员”筛选器, so that 我可以只看关于某一个特定对象的记录（如只看关于“男朋友”的）。
As a 回顾者, I want a 顶部有“时间”筛选器（支持过去一周、一个月、三个月、半年、一年、自定义）, so that 我能精准定位某个时间段的回忆。
As a 回顾者, I want a 点击时间线上的某张卡片进入详情页, so that 我能看到全尺寸的高清照片和完整的文字记录。
As a 回顾者, I want a 详情页底部显示精确到“秒”的系统记录时间, so that 我能回味这件美好事情发生的极其具体的时间点。
As a 期待惊喜的用户, I want a 在筛选出特定人员和时间范围后，点击一个“AI 总结报告”按钮, so that 我能获得基于这段时间记录的深度分析反馈。
As a 期待惊喜的用户, I want a AI 报告中包含这段时间的“关键词云”, so that 我能一眼看出最近的生活重心是什么。
As a 期待惊喜的用户, I want a AI 报告中包含“心情走势总结”, so that 我能了解自己在这段时间里的情绪起伏和状态。
As a 期待惊喜的用户, I want a AI 报告中包含一个“趣味人格侧写结论”（如：温柔细节控）, so that 我能获得一种被理解、被定义的专属情感体验。
As a 期待惊喜的用户, I want a AI 报告中包含“懂我的生活建议”, so that 我能感受到 AI 像一个贴心朋友一样关心我的日常。
As a 系统管理员, I want a 数据库底层支持多张图片存储的结构, so that 虽然第一版只允许用户传一张图，但未来升级多图功能时不需要重构数据库表。
Implementation Decisions
Architecture Choice: 使用 Next.js (App Router) 构建前后端一体化应用，利用 React Server Components 加速页面渲染，利用 Server Actions 处理表单提交（如保存记录）。部署在 Vercel 平台上以实现自动化 CI/CD。
BaaS Platform: 选用 Supabase 作为后端即服务（BaaS），统一处理 Auth（身份验证）、Postgres DB（结构化数据）和 Storage（多媒体文件）。
Deep Modules (深模块设计):
AuthService: 封装 Supabase Auth，暴露简单的 login, logout, getSession 接口，隐藏 JWT 令牌处理逻辑。
EventEngine: 处理核心业务逻辑的深模块，提供 createEvent, fetchEventsTimeline, getEventDetail 等接口。内部强制注入当前用户的 user_id，以保证严格的数据租户隔离。
StorageManager: 封装多媒体上传逻辑，对外暴露 uploadImage(file): string 接口，内部处理文件名哈希、上传进度和 public URL 获取。

模块设计：
AuthService (身份验证模块)：封装所有与 Supabase 邮箱密码登录相关的逻辑。
EventEngine (事件与人员核心模块)：处理小美好的增删改查。它内部会严格把控“数据隔离”，确保只返回当前登录用户的数据。
StorageManager (多媒体存储模块)：对外只提供一个 uploadImage(file) 接口，内部自动处理图片压缩和上传到 Supabase Storage 的复杂流程。
AISummaryEngine (AI 分析引擎 - 核心卖点)：对外只暴露 generateSummary(events) 接口。内部封装了 Prompt 拼接、调用大模型 API、解析输出 JSON 结构等复杂动作。

交互形式：
启动即“速记模式”（Landing/Edit State）
首屏展示： 一个大大的“今日小美好”卡片，顶部默认显示“记录给：[你/默认人员]”。
功能布局：
图片位： 一个大加号，点击直接唤起 iPhone 相机拍照或从相册选图。
文字区： “发生了什么？”（必填）+ “为什么觉得美好？”（选填）。
时间/人员切换： 底部小字显示“今天（可改）”，点击右上角可快速切换记录对象。
动作：
点击“记录成功”：卡片向上滑出，进入首页列表。
点击“取消”：直接进入首页列表。
首页“瀑布流”展示（Home/Timeline State）
显示单位： 以“天”为单位聚合。比如“2023年10月27日”下面挂着这一天记录的所有卡片。
筛选栏（吸顶）：
人选： 全部、你、爸爸、妈妈...
时间： 过去一周、一个月、自定义...
视觉效果： 每张卡片显示：照片缩略图 + 文字摘要 + 记录时间（如 14:30）。
点击反馈： 点击任意卡片，全屏进入详情页。
详情页（Detail State）
内容： 高清大图 + 完整内容 + 理由。
精确时间： 底部显示“记录于 2023-10-27 14:30:25”（满足你精确到秒的要求）。
操作： 支持编辑或删除。
第四部分：AI 深度总结（AI Analysis Logic）
当你女朋友在首页筛选出一段范围（比如“过去一个月”+“关于男朋友”）并点击 “AI 总结报告” 按钮时：
数据提取： 系统抓取该范围内的所有文字。
Loading 状态： 显示一个治愈的动画（如：AI 正在翻看你们的回忆...）。
生成报告（展示维度）：
✨ 关键词云： “冰淇淋、晚霞、电影”等。
🌈 心情天气： 总结这一个月是“多云转晴”还是“持续暖阳”。
🎭 人格侧写： AI 会给出一个称号，如“生活细节捕捉大师”。
💡 懂你的建议： “最近你记录了很多关于美食的小美好，下周末要不要去尝试那家新开的餐厅？”
